Backend Architecture / Incident
Kafka Consumer Scaling
280k message lag on the PaymentProcessor partition. Root cause analysis: single-threaded consumer combined with merchant tier hotspot (large merchants consuming disproportionate partition time). Fix: partition rebalance by tier + concurrent channel processing + idempotency-key dedup layer to prevent double-processing under concurrency. ADR-007 written documenting the trade-off analysis.

PostgreSQL Performance Optimisation

Fix: index tuning, query rewrite, CQRS read model separation so reporting queries hit a read replica with denormalised projections. Query time reduced 70%.
